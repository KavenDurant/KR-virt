# 集群管理模块 - 接口调用流程图

## 1. 集群管理主页面数据加载流程

```mermaid
graph TD
    A[用户进入集群管理页面] --> B[初始化组件状态]
    B --> C[默认激活概览标签页]
    C --> D[调用 loadTabData('overview')]
    D --> E[检查API锁状态]
    E -->|未锁定| F[设置API锁]
    E -->|已锁定| G[跳过重复调用]
    F --> H[调用 fetchClusterSummaryData]
    H --> I[发送 GET /cluster/summary]
    I --> J{API响应}
    J -->|成功| K[更新 clusterSummaryData]
    J -->|失败| L[设置错误状态]
    K --> M[清除API锁]
    L --> M
    M --> N[更新UI显示]
    
    O[用户切换到物理机列表] --> P[调用 loadTabData('list')]
    P --> Q[发送 GET /cluster/nodes]
    Q --> R[更新 realClusterData]
    
    S[用户切换到集群资源] --> T[调用 loadTabData('resources')]
    T --> U[发送 GET /cluster/resources]
    U --> V[更新 clusterResourcesData]
```

## 2. 主机详情页面数据加载流程

```mermaid
graph TD
    A[侧边栏选择主机] --> B[useSidebarSelection Hook 监听事件]
    B --> C[设置 sidebarSelectedHost 状态]
    C --> D[触发 useEffect 监听]
    D --> E[重置主机详情标签页到 basic]
    E --> F[清空之前的节点数据]
    F --> G[调用 fetchNodeDetailData]
    G --> H[发送 GET /node/summary?hostname=xxx]
    H --> I{API响应}
    I -->|成功| J[更新 nodeDetailData]
    I -->|失败| K[设置 nodeDetailError]
    J --> L[显示主机详情页面]
    K --> L
    
    M[用户切换到硬件信息标签] --> N{检查数据是否已加载}
    N -->|PCI数据未加载| O[调用 fetchNodePCIData]
    N -->|磁盘数据未加载| P[调用 fetchNodeDisksData]
    O --> Q[发送 GET /node/pci?hostname=xxx]
    P --> R[发送 GET /node/disks?hostname=xxx]
    Q --> S[更新 nodePCIData]
    R --> T[更新 nodeDisksData]
```

## 3. 主机操作流程

```mermaid
graph TD
    A[用户点击主机操作按钮] --> B{操作类型检查}
    B -->|进入维护模式| C[检查运行中虚拟机数量]
    B -->|其他操作| D[直接进入确认流程]
    C --> E{是否有运行中虚拟机}
    E -->|有| F[显示警告提示]
    E -->|无| D
    F --> G[操作终止]
    D --> H[显示操作确认对话框]
    H --> I{用户确认}
    I -->|取消| J[操作取消]
    I -->|确认| K[设置操作加载状态]
    K --> L[调用对应API]
    L --> M[发送操作请求]
    M --> N{API响应}
    N -->|成功| O[显示成功消息]
    N -->|失败| P[显示错误对话框]
    O --> Q[延迟2秒后刷新节点详情]
    P --> R[清除加载状态]
    Q --> R
```

## 4. 节点管理流程

```mermaid
graph TD
    A[用户点击添加节点] --> B[显示添加节点模态框]
    B --> C[用户填写节点信息]
    C --> D[表单验证]
    D -->|验证失败| E[显示验证错误]
    D -->|验证通过| F[提交表单]
    F --> G[调用 handleAddNode]
    G --> H[发送 POST /cluster/approve]
    H --> I{API响应}
    I -->|成功| J[显示成功消息]
    I -->|失败| K[显示错误消息]
    J --> L[关闭模态框]
    L --> M[刷新物理机列表]
    M --> N[触发侧边栏刷新事件]
    
    O[用户点击移除节点] --> P[设置待处理操作]
    P --> Q[显示安全确认模态框]
    Q --> R[用户输入确认文本]
    R --> S{确认文本正确}
    S -->|错误| T[禁用确认按钮]
    S -->|正确| U[启用确认按钮]
    U --> V[用户点击确认]
    V --> W[调用 executeRemoveNode]
    W --> X[发送 POST /cluster/remove]
    X --> Y{API响应}
    Y -->|成功| Z[显示成功消息并刷新]
    Y -->|失败| AA[显示错误消息]
```

## 5. 集群解散流程

```mermaid
graph TD
    A[用户点击解散集群] --> B[设置待处理操作为 dissolveCluster]
    B --> C[显示安全确认模态框]
    C --> D[显示解散集群警告信息]
    D --> E[用户输入确认文本 'dissolve cluster']
    E --> F{确认文本正确}
    F -->|错误| G[禁用确认按钮]
    F -->|正确| H[启用确认按钮]
    H --> I[用户点击确认]
    I --> J[调用 executeDissolveCluster]
    J --> K[发送 POST /cluster/dissolve]
    K --> L{API响应}
    L -->|成功| M[显示成功消息]
    L -->|失败| N[显示错误消息]
    M --> O[可能需要跳转到其他页面]
    N --> P[关闭确认模态框]
```

## 6. 侧边栏事件处理流程

```mermaid
graph TD
    A[侧边栏触发选择事件] --> B[useSidebarSelection Hook 监听]
    B --> C{事件类型判断}
    C -->|选择主机| D[设置 selectedHost 状态]
    C -->|选择虚拟机| E[设置 selectedVM 状态]
    C -->|选择集群| F[清空所有选择状态]
    D --> G[触发主机详情页面渲染]
    E --> H[触发虚拟机详情页面渲染]
    F --> I[显示集群管理主页面]
    
    J[侧边栏触发刷新事件] --> K[useSidebarRefresh Hook 监听]
    K --> L{刷新类型判断}
    L -->|cluster| M[调用 fetchRealClusterData]
    L -->|其他类型| N[忽略事件]
    
    O[侧边栏触发主机操作事件] --> P[useSidebarHostActions Hook 监听]
    P --> Q{操作类型验证}
    Q -->|有效操作| R[调用 handleNodeOperation]
    Q -->|无效操作| S[输出警告日志]
```

## 7. 错误处理流程

```mermaid
graph TD
    A[API调用开始] --> B[设置加载状态]
    B --> C[发送HTTP请求]
    C --> D{网络请求结果}
    D -->|网络错误| E[捕获异常]
    D -->|请求成功| F{API响应状态}
    F -->|success: true| G[处理成功数据]
    F -->|success: false| H[处理业务错误]
    E --> I[设置网络错误状态]
    H --> J[设置业务错误状态]
    I --> K[显示错误提示]
    J --> K
    G --> L[更新数据状态]
    K --> M[提供重试按钮]
    L --> N[清除加载状态]
    M --> O[用户可选择重试]
    N --> O
    O -->|重试| A
    O -->|不重试| P[保持错误状态]
```

## 8. 性能优化流程

```mermaid
graph TD
    A[组件初始化] --> B[设置防重复调用锁]
    B --> C[用户操作触发]
    C --> D{检查API锁状态}
    D -->|已锁定| E[跳过重复调用]
    D -->|未锁定| F[设置API锁]
    F --> G[执行API调用]
    G --> H[API调用完成]
    H --> I[清除API锁]
    
    J[用户快速切换标签] --> K[触发防抖定时器]
    K --> L[清除之前的定时器]
    L --> M[设置新的延迟执行]
    M --> N[延迟50-100ms后执行]
    N --> O[检查标签是否真正变化]
    O -->|未变化| P[跳过加载]
    O -->|已变化| Q[执行数据加载]
    
    R[硬件信息标签切换] --> S{检查数据是否已存在}
    S -->|已存在| T[直接显示缓存数据]
    S -->|不存在| U[按需加载数据]
    U --> V[缓存加载的数据]
```

---

*本流程图基于 KR-virt 集群管理模块的实际实现绘制，展示了主要的数据流和交互逻辑。*
